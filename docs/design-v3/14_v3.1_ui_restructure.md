# 14. v3.1 UI構造変更 — 医院中心ダッシュボード + サイト種別 + TOP管理

> v3設計からの構造変更を定義する

---

## 1. 変更概要

| 項目 | v3 | v3.1 |
|------|----|----|
| ダッシュボード単位 | ACMSシステム全体 | **1医院** |
| サイトの位置づけ | ≒医院 | 医院配下の**媒体の1つ** |
| サイト種別 | なし | HP / 専門サイト / 採用サイト / LP |
| トンマナ管理 | サイト単位のみ | **医院全体 + サイト固有**の二層 |
| TOPページ | 固定領域+slot | **セクション×コンポーネント管理**（下層と統一） |
| コンポーネント管理 | グローバル固定 | グローバル + **医院独自追加可能** |

---

## 2. エンティティ構造

```
Clinic (医院) ← ダッシュボードの単位
  ├── ClinicDesign (医院トンマナ: カラー・フォント・トーン)
  ├── ClinicComponent (医院独自コンポーネント定義)
  │
  ├── Site (HP)            site_type: hp
  │     ├── SiteDesign     (サイト固有デザイン、ClinicDesign継承+上書き)
  │     └── Page[]
  │           └── Section[] (セクション×コンポーネントタイプ)
  │
  ├── Site (専門サイト)     site_type: specialty
  │     ├── SiteDesign
  │     └── Page[]
  │
  ├── Site (採用サイト)     site_type: recruitment
  │     ├── SiteDesign
  │     └── Page[]
  │
  ├── StrategicTask[]       ← 医院レベルで管理
  ├── ImprovementReport[]
  └── FreeInputRequest[]
```

## 3. TOPページのセクション×コンポーネント管理

### テンプレートパターン（TOP用）

| パターン | 説明 | セクション構成例 |
|---------|------|----------------|
| `top_standard` | 標準HP向け | hero → features → treatments → greeting → blog → news → access |
| `top_specialty` | 専門サイト向け | hero → about → cases → flow → faq → cta |
| `top_recruitment` | 採用サイト向け | hero → message → culture → benefits → positions → faq |
| `top_minimal` | LP/仮サイト | hero → about → cta |

テンプレートを選ぶと**初期セクション構成が自動生成**され、そこから追加・削除・並び替えが可能。

### セクションのコンポーネントタイプ

各セクションに`component_type`を指定し、そのタイプに応じたHTML構造を持つ：

```json
{
  "section_id": "sec_01",
  "component_type": "hero_slider",
  "heading": "",
  "content_html": "<div class='com-hero com-hero--slider'>...</div>",
  "config": {
    "slides": 3,
    "autoplay": true
  },
  "lock_status": "unlocked",
  "order": 1
}
```

### TOPセクション用コンポーネントタイプ一覧

| タイプ | 説明 |
|--------|------|
| `hero_slider` | スライダーヒーロー |
| `hero_static` | 静的ヒーロー（1枚画像+テキスト） |
| `hero_video` | 動画ヒーロー |
| `features_grid` | 特徴グリッド（3〜4列） |
| `treatment_grid` | 診療メニューグリッド |
| `greeting` | 院長あいさつ |
| `latest_posts` | 最新ブログ/お知らせ |
| `case_gallery` | 症例ギャラリー |
| `flow_steps` | 来院の流れ |
| `access_map` | アクセス・地図 |
| `faq_accordion` | FAQ |
| `cta_banner` | CTAバナー |
| `freeform` | 自由HTML（従来の固定領域相当） |

## 4. デザイン継承の仕組み

```
グローバルデフォルト (design_tokens)
    ↓ 上書き
医院トンマナ (clinic_designs.tokens)
    ↓ 上書き
サイト固有 (site_designs.tokens)
    ↓ 上書き
カスタムCSS (site_designs.custom_css)
```

CSS生成時は4層をマージしてCSS Custom Propertiesに展開する。
