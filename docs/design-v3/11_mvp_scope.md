# 11. MVPスコープ

> 最初のリリースで何を作り、何を後回しにするかを明確に定義する

---

## 1. MVP目標

**1サイト（1医院）で、DNA-OS → 記事取り込み → 微調整 → 承認 → 公開 → ロールバック が一気通貫で動くこと。**

加えて：
- 戦略タスク→チャネルタスクの基本フローが動作
- フリー入力からのタスク生成が動作
- セクション単位の上書き制御が機能
- 例外コンテンツ（症例）の基本フローが動作

## 3. MVPで後回しにするもの

### 3-1. 後回し: 分析・データ基盤（L2）

| 機能 | 理由 | 代替手段 |
|------|------|---------|
| GA4連携 | 外部API連携の複雑さ | 手動でデータ入力 |
| Search Console連携 | 同上 | 手動 |
| KPI自動収集 | 同上 | 手動 |
| パフォーマンスダッシュボード | UI開発コスト | スプレッドシートで代替 |

### 3-2. 後回し: AI幕僚長の高度化

| 機能 | 理由 | MVP代替 |
|------|------|---------|
| 改善レポート自動生成 | AI連携の複雑さ | 手動レポート作成 |
| 高度な影響分析 | AI精度の検証が必要 | ルールベースマッピング |
| 自律的タスク生成 | 制御が難しい | 手動トリガー |
| 複数チャネル横断統制 | Webのみで十分 | — |

### 3-3. 後回し: 他チャネル

| 機能 | 理由 |
|------|------|
| GBPチャネル | Web確立後に着手 |
| SNSチャネル | 同上 |
| LINEチャネル | 同上 |

### 3-4. 後回し: 高機能UI

| 機能 | 理由 | MVP代替 |
|------|------|---------|
| メタ情報AI自動生成 | AI連携コスト | 手動入力 |
| 高機能ブログエディタ | 開発コスト | 基本HTMLエディタ |
| AIアシスト執筆 | AI連携コスト | 手動執筆 |
| 画像管理システム | 開発コスト | パス直接指定 |
| リアルタイムプレビュー | 開発コスト | 保存後プレビュー |
| 通知システム | COM-03 | 画面内バッジのみ |
| コンポーネントカタログUI | DSN-02 | 静的ドキュメント |

## 4. MVPのフェーズ分割

### Phase 1（Week 1-2）: データ基盤 + サイト/ページ管理

**目標**: v2のDBスキーマをv3に拡張し、基本CRUDが動作する

- [ ] v3マイグレーション作成（strategic_tasks, channel_tasks, sections対応等）
- [ ] モデル定義の更新/追加
- [ ] サイトCRUD（v2拡張）
- [ ] ページCRUD（v2拡張、content_classification, dna_source_key追加）
- [ ] ユーザー管理（v2そのまま）
- [ ] 認証（v2そのまま）

### Phase 2（Week 3-4）: コンテンツ取り込み + セクション管理

**目標**: DNA-OSの生成物を取り込み、セクション単位で管理できる

- [ ] コンテンツ取り込みサービスの拡張（セクション自動パース）
- [ ] セクション一覧/編集画面
- [ ] セクションロック制御
- [ ] 世代比較（セクション単位差分）
- [ ] 微調整（HTMLエディタ + 差分記録）
- [ ] プレビュー機能

### Phase 3（Week 5-6）: 戦略タスク + 承認 + 公開

**目標**: タスクフロー（戦略→チャネル→承認→公開）が一気通貫で動く

- [ ] DNA-OS変更検出（ポーリング）
- [ ] 影響マッピング（ルールベース）
- [ ] 戦略タスクCRUD
- [ ] チャネルタスク自動分解
- [ ] フリー入力→AI解釈（Claude API）
- [ ] 承認フロー（承認待ち一覧、承認詳細）
- [ ] ビルドエンジン拡張（セクション結合対応）
- [ ] FTPデプロイ（v2そのまま）
- [ ] ロールバック（v2そのまま）

### Phase 4（Week 7-8）: 例外コンテンツ + 統合テスト

**目標**: 例外コンテンツが動作し、1サイトで全フローが通る

- [ ] 症例CRUD（構造化入力フォーム）
- [ ] コンプライアンス基本チェック
- [ ] 二段階承認フロー
- [ ] 全導線の結合テスト（P1〜P7）
- [ ] 1サイト実証（亀有矯正歯科）
- [ ] バグ修正 + UX改善

---

## 5. MVP完了基準

以下がすべて動作することを確認：

1. **P1**: DNA-OS更新 → タスク生成 → Web反映 → 承認 → 公開
2. **P3**: フリー入力 → AI解釈 → タスク化 → 実行 → 公開
3. **P4**: 新規ページ → 取り込み → プレビュー → 公開
4. **P6**: ブログ下書き → チェック → 承認 → 公開
5. **P7**: 症例作成 → コンプライアンスチェック → 二段階承認 → 公開
6. セクションロックした箇所がAI再生成でスキップされること
7. ロールバックで前の状態に戻せること
8. 1サイト分のFTPデプロイが成功すること
