# 10. 画面遷移

> 戦略ダッシュボードからWeb公開までの自然な遷移を定義する

---

## 1. ナビゲーション構造

```
[グローバルナビ]
├── ダッシュボード (COM-02)
├── 戦略 ▾
│   ├── 戦略概要 (STR-01)
│   ├── 改善レポート (STR-02)
│   ├── タスク一覧 (STR-04)
│   ├── 修正依頼 (STR-06)
│   └── DNA-OS更新 (STR-07)
├── サイト ▾
│   ├── サイト一覧 (SITE-01)
│   └── [サイト名] → サイト詳細 (SITE-03)
├── 承認 (APR-01)  ※バッジで件数表示
├── デザイン ▾
│   ├── トークン (DSN-01)
│   └── コンポーネント (DSN-02)
└── 管理 ▾
    └── ユーザー (USR-01)

[サイトコンテキストナビ] ※サイト選択後に表示
├── ページ一覧 (PAGE-01)
├── 取り込み (IMP-01)
├── 例外コンテンツ (EXC-01)
├── 公開 (PUB-01)
└── デザイン設定 (DSN-03)
```

---

## 2. 主要フロー: 戦略→Web公開

### フロー2-1: DNA-OS更新→Web公開

```
STR-07 DNA-OS更新差分確認
  │ 変更内容の確認
  │ [タスク生成する] ボタン
  ▼
STR-05 戦略タスク詳細
  │ 生成されたタスクの確認
  │ [承認する] ボタン
  ▼
STR-08 チャネル実行状況
  │ チャネルタスクの進捗確認
  │ [Web管理へ] リンク
  ▼
PAGE-03 ページ詳細
  │ 修正案の確認
  │ [差分を見る] リンク
  ▼
PAGE-06 世代比較
  │ 変更前後の差分確認
  │ [微調整する] or [承認する]
  ├──────────────────┐
  ▼                  ▼
PAGE-05           APR-02
セクション編集     承認詳細
  │                  │
  │ [保存]           │ [承認]
  ▼                  ▼
PAGE-03           PUB-01
ページ詳細         公開ダッシュボード
                    │
                    │ [デプロイ]
                    ▼
                  PUB-02
                  デプロイ確認
                    │
                    │ [実行]
                    ▼
                  PUB-03
                  デプロイ履歴
```

### フロー2-2: フリー入力→Web公開

```
STR-06 フリー入力修正依頼
  │ テキスト入力
  │ [送信]
  ▼
STR-06 (AI解釈結果表示)
  │ 解釈確認
  │ [この解釈で合っている]
  ▼
STR-05 戦略タスク詳細
  │ 自動生成されたタスクの確認
  │ [承認する]
  ▼
(以降フロー2-1と同じ)
```

### フロー2-3: 新規ページ生成

```
STR-06 or PAGE-01
  │ 新規ページ生成依頼
  ▼
STR-05 戦略タスク詳細
  │ [承認する]
  ▼
(記事生成パイプライン実行 - バックグラウンド)
  ▼
IMP-01 取り込みフォーム
  │ 生成結果のURL入力 or 自動取得
  │ [取り込み]
  ▼
IMP-02 取り込み結果確認
  │ セクション分割結果の確認
  │ [保存]
  ▼
PAGE-03 ページ詳細
  │ プレビュー確認
  │ [公開へ進む]
  ▼
(フロー2-1の承認→公開と同じ)
```

### フロー2-4: ブログ執筆→公開

```
PAGE-01 ページ一覧 (blog)
  │ [新規ブログ作成]
  ▼
PAGE-02 ページ作成
  │ タイトル、slug入力
  │ [作成]
  ▼
PAGE-05 セクション編集
  │ ブログ本文の入力
  │ AIアシストボタン
  │ [保存]
  ▼
PAGE-03 ページ詳細
  │ 品質チェック結果表示
  │ [公開申請]
  ▼
APR-02 承認詳細
  │ 管理者が確認
  │ [承認]
  ▼
PUB-01 公開ダッシュボード
  │ [デプロイ]
  ▼
PUB-02 → PUB-03
```

### フロー2-5: 症例作成→公開

```
EXC-01 例外コンテンツ一覧
  │ [新規症例作成]
  ▼
EXC-02 症例作成
  │ 構造化データ入力
  │ 画像アップロード
  │ [保存]
  ▼
EXC-03 コンプライアンスチェック結果
  │ 自動チェック結果の確認
  │ 問題があれば修正
  │ [一次承認へ進む]
  ▼
EXC-04 例外コンテンツ承認
  │ 管理者が一次承認
  │ [一次承認]
  ▼
EXC-04 例外コンテンツ承認 (最終)
  │ 院長/法務が最終承認
  │ [最終承認]
  ▼
PUB-01 → PUB-02 → PUB-03
```

## 3. 画面間の接続ポイント

### 戦略→Web管理の接続

| 起点 | 遷移先 | トリガー |
|------|--------|---------|
| STR-05 戦略タスク詳細 | PAGE-03 ページ詳細 | チャネルタスクの対象ページをクリック |
| STR-08 チャネル実行状況 | PAGE-03 ページ詳細 | タスクの対象ページをクリック |
| STR-07 DNA-OS更新差分 | PAGE-06 世代比較 | 影響ページの「差分を見る」をクリック |
| STR-04 タスク一覧 | APR-01 承認待ち | 承認待ちフィルタを適用 |

### Web管理→戦略の接続

| 起点 | 遷移先 | トリガー |
|------|--------|---------|
| PAGE-03 ページ詳細 | STR-05 戦略タスク詳細 | 関連タスクをクリック |
| PUB-03 デプロイ履歴 | STR-08 チャネル実行状況 | タスクIDをクリック |
| APR-01 承認待ち | STR-05 戦略タスク詳細 | 戦略タスク承認をクリック |

## 4. ステータス遷移図

### 戦略タスクのステータス遷移

```
                    ┌─────────┐
                    │  draft  │
                    └────┬────┘
                         │ AI幕僚長がタスク生成
                         ▼
                ┌──────────────────┐
                │ pending_approval │
                └───────┬──┬──────┘
           承認 │       │ キャンセル
                ▼       ▼
          ┌──────────┐  ┌───────────┐
          │ approved │  │ cancelled │
          └────┬─────┘  └───────────┘
               │ チャネルタスク配信
               ▼
         ┌─────────────┐
         │ in_progress │
         └──────┬──────┘
                │ 全チャネルタスク完了
                ▼
          ┌───────────┐
          │ completed │
          └───────────┘
```

### チャネルタスクのステータス遷移

```
          ┌─────────┐
          │ pending │
          └────┬────┘
               │ 実行開始
               ▼
        ┌─────────────┐
        │ in_progress │
        └──────┬──────┘
               │ 修正案完成
               ▼
        ┌──────────────┐
        │ review_ready │◄──── rejected (再実行)
        └──────┬───────┘
               │ 承認
               ▼
          ┌──────────┐
          │ approved │
          └────┬─────┘
               │ デプロイ
               ▼
          ┌──────────┐
          │ deployed │
          └────┬─────┘
               │ 結果確認
               ▼
         ┌───────────┐
         │ completed │
         └───────────┘
```

### ページ世代のステータス遷移

```
          ┌───────┐
          │ draft │ ← 取り込み直後 / 再生成直後
          └───┬───┘
              │ セクション確認完了
              ▼
          ┌───────┐
          │ ready │ ← 微調整完了 / チェック完了
          └───┬───┘
              │ 承認
              ▼
        ┌──────────┐
        │ approved │
        └────┬─────┘
             │ デプロイ
             ▼
       ┌───────────┐     ┌────────────┐
       │ published │────►│ superseded │ ← 次の世代がpublishedになった時
       └───────────┘     └────────────┘
              ▲
              │ ロールバック
              │
       ┌────────────┐
       │ superseded │
       └────────────┘
```

### 例外コンテンツのステータス遷移

```
        ┌───────┐
        │ draft │
        └───┬───┘
            │ 公開申請
            ▼
     ┌──────────────┐
     │ first_review │
     └──────┬───────┘
            │ 一次承認
            ▼
     ┌──────────────┐     ┌──────────┐
     │ final_review │────►│ rejected │
     └──────┬───────┘     └──────────┘
            │ 最終承認
            ▼
       ┌──────────┐
       │ approved │
       └────┬─────┘
            │ デプロイ
            ▼
      ┌───────────┐     ┌──────────┐
      │ published │────►│ archived │
      └───────────┘     └──────────┘
```

## 5. 通知トリガー

| イベント | 通知先 | 通知内容 |
|---------|--------|---------|
| DNA-OS変更検出 | admin, editor | 「DNA-OSが更新されました。確認してください」 |
| 戦略タスク生成 | admin | 「新しい戦略タスクが生成されました」 |
| 承認依頼 | 承認者 | 「承認待ちがN件あります」 |
| チャネルタスク完了 | editor | 「タスク[ID]の修正案が完成しました」 |
| デプロイ成功 | admin, editor | 「[サイト名]のデプロイが完了しました」 |
| デプロイ失敗 | admin | 「[サイト名]のデプロイが失敗しました」 |
| タスク滞留 | admin | 「[ID]が24時間以上停滞しています」 |
| コンプライアンス警告 | admin | 「例外コンテンツにコンプライアンス問題があります」 |
