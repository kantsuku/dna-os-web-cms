# ACMS 設計 v2 — 全体概要

## ACMSとは

AI Content Management System。
AIがコンテンツを生成・管理し、人間は「出す/止める」と微細な調整だけを行うシステム。

## 従来CMSとの違い

| | 従来CMS | ACMS |
|---|---|---|
| コンテンツ作成 | 人間が書く | AIが生成する |
| 承認フロー | 下書き→レビュー→承認→公開 | AIが生成→人間が公開ボタンを押す |
| 編集 | 主たる作業 | 例外的な微修正（差分記録付き） |
| テンプレート | 人間がページごとに組む | AIが構造化済みHTMLを出し、デザインシステムと自動結合 |
| 更新 | 人間が更新する | DNA-OSの原本が変わればAIが再生成 |

## 3つの責務分離（v2）

```
DNA-OS              = 原本管理（医院の人格）
clinic-page-generator = コンテンツ生成（AIが書く）
ACMS                 = 公開統制（受け取り・デザイン結合・公開・世代管理）
```

### 原則
1. **コンテンツはAIが作る** — ACMSはコンテンツを「作る場所」ではなく「流す場所」
2. **人間はデザインとGo/NoGoを握る** — コンテンツの中身はAI、見た目と公開判断は人間
3. **差分は全て記録する** — AI生成版と人間修正版の差分を常にトレース可能に
4. **世代管理でロールバック** — いつでも前の状態に戻せる
5. **デザインシステムとコンテンツは分離** — 同じコンテンツに別デザインを当てられる
