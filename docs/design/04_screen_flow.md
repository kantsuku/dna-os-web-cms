# 4. 画面遷移

## メインフロー（日常運用）

```
ログイン
  │
  ├─[admin/editor]──→ ダッシュボード (A1)
  │                      │
  │                      ├──→ サイト一覧 (B1)
  │                      │      └──→ サイト詳細 (B2)
  │                      │             ├──→ ページ一覧 (C1)
  │                      │             │      └──→ ページ詳細 (C2)
  │                      │             │             └──→ セクション管理 (D1)
  │                      │             │                    ├──→ 微調整画面 (D3)
  │                      │             │                    └──→ 差分比較 (D4)
  │                      │             ├──→ 公開確認 (F1)
  │                      │             │      └──→ プレビュー (F2)
  │                      │             ├──→ 公開履歴 (F3)
  │                      │             │      └──→ ロールバック確認 (F4)
  │                      │             └──→ 例外コンテンツ一覧 (G1)
  │                      │                    └──→ 例外コンテンツ詳細 (G2)
  │                      │
  │                      └──→ 承認待ち一覧 (E1)
  │                             └──→ 承認詳細 (E2)
  │
  └─[client]──→ クライアントダッシュボード (H1)
                   ├──→ ブログ投稿画面 (H2)
                   └──→ 投稿一覧・編集 (H3)
```

## コンテンツ公開フロー（ステータス遷移）

```
DNA-OS原本
    │
    ▼ 同期
[draft] ────────────────→ セクションに差し込み
    │                         │
    │                    微調整が必要？
    │                    ├─ No → そのまま
    │                    └─ Yes → 微調整画面 (D3)
    │                              │
    │                              ▼
    │                         差分保存
    │                              │
    ▼                              ▼
[pending_review] ←─── 公開申請ボタン
    │
    ▼
承認待ち一覧 (E1) → 承認詳細 (E2)
    │
    ├─ 承認 → [approved]
    │            │
    │            ▼
    │       公開確認画面 (F1)
    │            │
    │            ├─ プレビュー確認 (F2)
    │            │
    │            ▼
    │       デプロイ実行
    │            │
    │            ▼
    │       [published] ──→ 公開履歴 (F3)
    │
    ├─ 却下 → [draft] に戻る（修正理由付き）
    │
    └─ 差し戻し → [draft] に戻る（コメント付き）
```

## 再生成・同期フロー

```
DNA-OSで原本更新
    │
    ▼
CMS同期実行（手動 or 定期）
    │
    ▼
セクションの override_policy を確認
    │
    ├─ auto_sync
    │    → 新バリアント自動作成、旧バリアントは superseded
    │    → status: draft（承認フローに再度乗る）
    │
    ├─ confirm_before_sync
    │    → 差分比較画面 (D4) に候補として表示
    │    → editor が採用/棄却を判断
    │
    ├─ manual_only
    │    → 通知のみ。「原本が更新されました」
    │    → editor が手動で取り込むか判断
    │
    └─ locked
         → 何もしない。例外コンテンツ等
```

## クライアント投稿フロー

```
クライアントダッシュボード (H1)
    │
    ▼
ブログ投稿画面 (H2)
    │
    ▼
[下書き保存] or [公開申請]
    │
    ├─ 下書き → 投稿一覧 (H3) で管理
    │
    └─ 公開申請 → 承認待ち一覧 (E1) に乗る
                     │
                     ▼
                  承認 → 自動デプロイ or 次回公開バッチに含める
```
