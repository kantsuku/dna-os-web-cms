## 技術スタック

| レイヤー | 技術 | 理由 |
|---|---|---|
| CMS バックエンド | Laravel 11 (PHP 8.2+) | XServer互換、承認フロー・権限管理が組みやすい |
| CMS フロント | Livewire + Alpine.js | SPA不要、Laravelと密結合で開発速度重視 |
| DB | MySQL 8.0 | XServer標準、Laravel標準 |
| 公開サイト | PHP テンプレート + 静的アセット | XServer上でそのまま動く。重い処理なし |
| デプロイ | FTP (phpseclib) | XServer標準アクセス。SSH使えるプランなら SSH優先 |
| DNA-OS連携 | HTTP (GAS WebApp URL) | 既存APIをそのまま利用 |
| キュー/ジョブ | Laravel Queue (database driver) | ビルド・デプロイ・同期の非同期処理 |

## 3つの明確な責務分離

```
DNA-OS        = 原本管理 + AI構造化（ここが真実の源泉）
Web CMS       = 公開統制（組み立て・微調整・承認・公開・バージョン管理）
公開サイト     = 表示のみ（ロジックを持たない）
```

### 原則
1. **原本はDNA-OS** — CMSは原本を書き換えない
2. **CMSは差分を持つ** — 微調整内容はCMS側に「差分」として保存。原本は汚さない
3. **公開サイトはダム** — CMSがビルドした成果物を配置するだけ。公開サイト側にCMSロジックは置かない
4. **上書き制御はCMSの責務** — 再生成と人間修正の衝突防止はCMSが一元管理
